Project structure

Project follows the module structure proposed by nrwl. There is one app, called 'people', and libs, used under this app.

'People' app contains only a routing module. This routing module lazy loads given libs based on path in URL.

There are 3 main grouping folders used for libs:

  • person - used under '/person' URL
  • specification - used under '/specification' URL
  • shared - containing dummy components, resources etc. used by all libs

Person library

Contains a feature module with routing under 'person' path, and a data access module, used to communicate with API. Routing module is placed in feature library. 'Person' smart component is also responsible for opening a dialog with person details after entering an ID in URL. Dummy components, used by this feature library, are placed in person list and details dialog.

Specification library

Contains a feature module with routing under 'specification' path, and a smart component with specification of people app.

Shared library

Contains a shared navbar component which is used in app root component, to navigate between person list and specification.

Data access module

Contains a data service used to get person collection and person details data. It also provides a cache mechanism, based on comparing timestamp of last request with the new one.

Person list

Dummy component with onPush change detection strategy. Data is passed only by input and output properties. Contains an Angular Material table with sort header. Displays a loading indicator or error message, depending on data from parent.

Person details dialog

Dummy component with onPush change detection strategy. Data is passed by injecting an object of properties from parent when opening a dialog. Displays a loading indicator or error message, depending on result of loading given person.
